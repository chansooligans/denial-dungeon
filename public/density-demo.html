Source-asset density / canvas resolution comparison

All three sprites display at the same visual size (~256px tall) so the comparison is purely about how many art-pixels go into the figure. 32×32 is the current production sprite, ported verbatim from BootScene.makePlayerDirection. 64×64 and 128×128 are sketches — same character, but with extra detail strokes that can only exist at higher density. Real production assets would need a proper pass; these are just to show the ceiling.

Canvas resolution: 960×640 vs 1920×1280

Both panels show the same logical scene (top-down hospital corridor with player and an NPC). The 960×640 panel is the current canvas resolution; 1920×1280 doubles it. Sprite scale unchanged in pixels, so the larger canvas exposes more world per screen and finer positional detail (and bigger text headroom). Each is shown at a fixed CSS width, so the larger one is downscaled — meaning on a retina screen, the 1920×1280 backing store gives crisper edges even when the displayed size matches.
