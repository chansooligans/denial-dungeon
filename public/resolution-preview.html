Resolution preview

Same tile layout, two render scales. Shows what the hospital looks like at the current TILE = 32 game-pixel grid (left) versus an upgraded TILE = 64 grid (right). Objects pulled live from public/sprites/objects-raw/; characters from player/ and npcs-raw/. Both panels display the same world coordinates, so the right panel is literally twice as wide / tall in screen pixels.

At 32: objects are downsampled from their 64×64 LoRA source via setDisplaySize(32, 32) — fits the existing tile grid but loses detail. At 64: objects render at native source resolution, full LoRA detail visible. Characters are 64-px native at both scales, but they read more naturally at 64-tile because they fit one tile instead of overflowing into the tile above.
